# audience: operations

統合仕様候補を、運用管理者向けに変換してください。

重視すること:

- 実際に使われている endpoint / 機能候補
- 4xx/5xx や拒否系の観測パターン
- master data や設定が効いている制約
- 運用上注意すべき例外処理
- 観測期間、観測件数、未観測の限界

コード詳細や SQL 全文は必要な場合だけ短く触れてください。
count=1、rare pattern、短時間の観測期間は必ず低観測として明記してください。

4xx/5xx や拒否系は、観測された status と custom event を根拠にしつつ、運用原因を断定しないでください。master data や設定が原因と思われる場合も「候補」として書き、確認方法を添えてください。
