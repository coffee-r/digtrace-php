# audience: planning

統合仕様候補を、企画担当向けに変換してください。

重視すること:

- 現在できること / できないこと
- 現行制約
- 制約の根拠候補
- 変更した場合に影響しそうな領域
- 企画上確認すべき前提

実装詳細ではなく、意思決定に必要な制約と確認論点を中心にしてください。
各制約には根拠レベルを添えてください。

- **観測で確認**: trace/export/diff に直接ある
- **仕様候補**: 観測と実装が対応している
- **推論**: 業務上の意味づけ
- **未確認**: 追加観測や企画/運用確認が必要

custom event 名や endpoint 名をそのまま企画用語として断定しないでください。count=1 は低観測として扱い、変更判断の前に確認が必要な論点として書いてください。
